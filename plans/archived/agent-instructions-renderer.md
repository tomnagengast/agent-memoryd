# Agent Instructions Renderer

> updated: 2026-06-16T11:56:00-07:00

## Goal

Build a cross-harness instruction system that works for Codex, Claude Code, OpenCode, Amp, and future coding agents without requiring the agent to spend tool calls manually reading secondary instruction files.

The core idea: keep one canonical source tree of instruction fragments, then deterministically render harness-specific autoload files before launching each agent.

## Problem

Agent harnesses differ in how they load persistent instructions:

- Codex autoloads `AGENTS.md`, but does not expand include-style lines like `@./SYSTEM.md`.
- Claude Code has `CLAUDE.md` plus startup hooks that can inject session context.
- OpenCode uses `AGENTS.md`.
- Amp supports `AGENTS.md` and historically used `AGENT.md`.

Because include/import behavior is not portable, `AGENTS.md` / `CLAUDE.md` / `AGENT.md` should be treated as compiled outputs, not source files.

## CLI

```sh
ai-instructions render --target codex
ai-instructions render --target claude
ai-instructions render --target opencode
ai-instructions render --target amp
ai-instructions render --all
ai-instructions check --all
```

`render` writes the autoload file for each harness. `check` exits nonzero when rendered output is stale.

## Rendered Output Format

Each generated file should include provenance and fragment boundaries:

```md
<!--
Generated by ai-instructions render --target codex.
Do not edit directly. Edit files under:
$CMPTR_HOME/config/ai/instructions/
-->

<!-- BEGIN fragments/base.md -->
...
<!-- END fragments/base.md -->

<!-- BEGIN harness/codex.md -->
...
<!-- END harness/codex.md -->
```

## Implementation Sketch

```python
#!/usr/bin/env python3
import argparse
import os
import tomllib
from pathlib import Path

ROOT = Path(os.environ.get("CMPTR_HOME", Path.home() / "cmptr")) / "config/ai/instructions"
MANIFEST = ROOT / "manifest.toml"


def expand(path: str) -> Path:
    return Path(os.path.expandvars(path)).expanduser()


def render_target(name: str, cfg: dict) -> str:
    chunks = [
        "<!--",
        f"Generated by ai-instructions render --target {name}.",
        "Do not edit directly. Edit config/ai/instructions fragments.",
        "-->",
        "",
    ]

    for rel in cfg["fragments"]:
        path = ROOT / rel
        if not path.exists():
            raise FileNotFoundError(path)
        chunks.append(f"<!-- BEGIN {rel} -->")
        chunks.append(path.read_text())
        chunks.append(f"<!-- END {rel} -->")
        chunks.append("")

    return "\n".join(chunks).rstrip() + "\n"


def write_atomic(path: Path, content: str) -> bool:
    path.parent.mkdir(parents=True, exist_ok=True)
    old = path.read_text() if path.exists() else None
    if old == content:
        return False

    tmp = path.with_suffix(path.suffix + ".tmp")
    tmp.write_text(content)
    tmp.replace(path)
    return True


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("command", choices=["render", "check"])
    parser.add_argument("--target")
    parser.add_argument("--all", action="store_true")
    args = parser.parse_args()

    manifest = tomllib.loads(MANIFEST.read_text())
    targets = manifest["targets"]

    names = list(targets) if args.all else [args.target]
    if not names or names == [None]:
        raise SystemExit("pass --target <name> or --all")

    changed = False
    for name in names:
        cfg = targets[name]
        content = render_target(name, cfg)
        output = expand(cfg["output"])

        if args.command == "check":
            if output.exists() and output.read_text() == content:
                print(f"{name}: ok")
            else:
                print(f"{name}: stale")
                changed = True
        else:
            did_change = write_atomic(output, content)
            print(f"{name}: {'updated' if did_change else 'unchanged'}")
            changed = changed or did_change

    raise SystemExit(1 if args.command == "check" and changed else 0)


if __name__ == "__main__":
    main()
```

## Launch Adapters

Each agent launcher should render before exec:

```sh
agent-codex() {
  ai-instructions render --target codex
  exec codex "$@"
}

agent-claude() {
  ai-instructions render --target claude
  exec claude "$@"
}

agent-opencode() {
  ai-instructions render --target opencode
  exec opencode "$@"
}

agent-amp() {
  ai-instructions render --target amp
  exec amp "$@"
}
```

## Design Rules

- One canonical source tree.
- Many generated harness autoload files.
- No dependency on non-portable include syntax.
- No agent turn spent reading instruction fragments.
- Renderer stays deterministic and boring: concatenate first, template features later only if needed.
- Generated files should be safe to diff and should not change when fragment contents have not changed.

## Open Questions

- Should generated files be committed, or should they be ignored and generated during shell/launcher setup?
- Should the renderer also generate symlinks for harnesses that support the same target filename?
- Should `check --all` run in dotfile CI or pre-commit?
- Should Claude Code continue to use SessionStart hook injection for dynamic context, separate from this static compiled instruction system?
